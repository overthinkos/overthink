# Changelog — 2026-07

### 2026-07-01 — feat(charly): externalize the migrate chain into candy/plugin-migrate via host-prelift (TRACK 2 / M15)

The schema-migration chain (~49 migrators + the registry, ~14.3K LOC) moves out of charly core into a new
COMPILED-IN candy; `charly/migrate.go` becomes a thin in-core shim. Unlike egress/k8sgen (pure transforms), the
migrate chain is LOADER-COUPLED — 5 migrators call core loader functions (`LoadUnified`, `LoadBundleConfig`,
`embeddedDefaults`, `VerbCatalog`) and manipulate package-main-only loader types a candy can't import. Resolved
via the operator-approved HOST-PRELIFT pattern.

- New `candy/plugin-migrate` (compiled-in, `verb:migrate`, serving `OpRun`): the registry
  (`MigrationStep`/`migrationSteps`/`runMigrations`, which now also returns the changed-step list) + the ~44
  loader-independent migrators + their tests, moved verbatim.
- **Host-prelift** for the 5 loader-coupled migrators: the in-core shim runs the loader-coupled lookups
  host-side and passes pre-resolved inputs (image names, local templates, bundle-volume summary, the
  build-vocabulary key set / host-deploy path) into the plugin's `OpRun`; those migrators consume the
  pre-resolved inputs instead of calling the loader — behavior preserved (the ported migrate tests pass). The
  act-verb set + credential store are injected at startup (constants/host singletons), not per-run.
- R3 relocations to `charly/plugin/kit` (one copy, both repos import): `CalVer`/`ParseCalVer`/`MustCalVer`/
  `LatestSchemaVersion`, `MigrateContext`/`MigrateReply`, `PluralToSingularYAMLKeys`, `LedgerSchemaVersion`, the
  path consts + shared yaml/file helpers (`FileExists`/`FirstNonEmpty`/`MapValue`/`FirstYAMLVersionLine`/…); the
  step-keyword consts; core aliases them via `kit_aliases.go`. `RejectLegacyPluralKeys` relocated to a
  non-migrate core file (it stays in core — a load-time gate).
- In-core shims (mirror egress.go/k8s_generate.go): `charly/migrate.go` (`RunMigrations`/`RunProjectMigrations`
  resolve `verb:migrate` + prelift + `Invoke(OpRun)`); `migrate_cmd.go` + `refs.go` consume the shim unchanged.
- R5: the moved `migrate_*.go` chain deleted from core; the loader-coupled migrate tests relocated to core
  (`charly/migrate_*_core_test.go`, importing the candy); `compiled_plugins:` + plugin-migrate; the stale
  `/charly-build:migrate` registry table extended to HEAD (2026.174.1100) + the go/cutover-policy skill refs.
