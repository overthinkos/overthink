### 2026-07-01 — docs: C13/M15 complete; RETRACT C13b (OpBootstrap auto-migrate-at-load) as unsound

C13a (the 2026-07-01 M15 entry) externalized the migrate chain to `candy/plugin-migrate` (serving `OpRun`).
The M15 entry promised a follow-up "C13b" that would make plugin-migrate a **bootstrap-phase** plugin serving
`OpBootstrap`, so a stale config's root bytes would auto-migrate in-memory at load via the F9 `runBootstrapPhase`
(the rev.3 vision: "the kernel reads raw `charly.yml` bytes → the migrate plugin returns the upgraded config").
RDD on the live code refuted that design, so C13b is RETRACTED and C13/M15 is declared COMPLETE as landed.

Why C13b is architecturally unsound (ground truth from the live code):

- The migrate chain is **whole-project file-based + host-coupled**, not a raw-root-bytes transform: `runDocMigration`
  scans candidate files across the whole project dir and rewrites each on disk; `rewriteDocFile` writes specific
  files + `.bak` backups; `migrateHostOverlayDoc` mutates `~/.config/charly/charly.yml` host state.
- `MigrateContext` needs **host-prelifted inputs** (`ImageNames` from `LoadUnified(dir)`, `BundleVolumes` from
  `LoadBundleConfig`, …) — produced by a *completed* `LoadUnified`. But `runBootstrapPhase` fires *inside*
  `LoadUnified`, before the parse → a re-entry cycle.
- `runBootstrapPhase` transforms **only the root bytes**; a stale project's candy files / siblings stay stale and
  fail the `#NodeDoc` / per-entity gates regardless. A partial in-memory pass would also let the trailing
  `calver-schema` stamp write **HEAD onto an incompletely-migrated doc** — masking incompleteness.

Resolution (operator-approved): keep the documented, safe model — the load gate REJECTS a stale config with
`Run: charly migrate`, and `charly migrate` (+ remote-cache auto-migration) runs the full file-based chain
explicitly (with backups). The F9 bootstrap-phase machinery stays as foundational infra, proven by the no-op
`candy/plugin-example-bootstrap` + the F9 tests; it simply has no production consumer yet.

R5 claim-keyed sweep (this commit): corrected every `.go` comment + the M15 CHANGELOG note that asserted migrate
(or egress) is/will-be a bootstrap-phase auto-migrate plugin — `charly/bootstrap_phase.go`, `charly/unified.go`,
`charly/plugin_bootstrap_test.go`, `charly/plugin/sdk/phase.go`, `candy/plugin-example-bootstrap/plugin.go`, and
the 2026-06 + 2026-07 CHANGELOG forward-claims; the `/charly-internals:go` skill is swept in the plugins repo.
